# 협상제안서 시스템 작업 기준 v3.0

## 문서 생성 기준

### 입력 매개변수
| 매개변수 | 설명 | 필수 |
|:---|:---|:---:|
| `--dong` | 동 번호 (예: 410) | O |
| `--ho` | 호 번호 (예: 1701) | O |
| `--asking-price` | 매도자 호가 (만원, 예: 92000) | O |
| `--comp-price` | 동일유사매물 호가 (만원, 예: 94000) | O |
| `--stage1-price` | 사용자 지정 1차 제안가 (만원, 예: 91000) | O |
| `--type` | 거래 유형 (매매/전세/월세, 기본: 매매) | - |
| `--pdf` | PDF 동시 생성 | - |

### 실행 명령
```
python generate_proposal.py --dong 410 --ho 1701 --asking-price 92000 --comp-price 94000 --stage1-price 91000 --pdf
```

## 문서 구조 (모든 세대 공통 적용)

### 1페이지: 표지
- `cover_page.png` 이미지 사용 (협상제안서_출력 폴더)
- TOP SECRET 비밀문서 스타일

### 2페이지: 매물 정보
- 24px 중앙정렬 일반 텍스트
- 대상 세대, 평형, 층수, 등급, 호가, 유사호가, 제안가 표시

### 3~7페이지: 5단계 제안
- **1차 제안**: 사용자 입력 가격 기반 (데이터+심리학)
- **2차 제안**: 1차+1%, 조건부 양보 (상호성 원리)
- **3차 제안**: 1차+2%, 수용+대안 조건 (프레이밍)
- **4차 제안**: 마지노선=호가 (손실회피, 보유비용)
- **5차 대안**: 돌발대비 Walk-away 전략

### 부록: 지원 데이터
- A. 실거래 전체 이력 + ASCII 차트
- B. 비교 단지 시세
- C. 한국부동산원 시세
- D. 현재 시장 지표 (KB)
- E. 단지 인프라 호재
- F. 옵션 시장 가치
- G. 학술 논문 기반 심리학 8대 원리 + 6단계 가이드

## 데이터 수집 기준
- 면적: 공급면적 기준 통일 (수집기준.md 참조)
- 기간: 특정기간 미지정 시 **최근 실거래가 적용** (기간 제한 없이 가장 최근 거래 우선)
- 유사거래 비교법: 최근 + 유사 층수 거래 우선 적용
- 출처: 국토교통부, 네이버 부동산, 아실
- 공급/전용 구분 반드시 확인 후 수집

## 가격 파생 로직
- 1차 = 사용자 입력 (`--stage1-price`)
- 2차 = 1차 × 1.01
- 3차 = 1차 × 1.02
- 4차 마지노선 = 호가 (`--asking-price`)

## 심리학 원리 (각 단계별 필수 포함)
1. 앵커링 (Tversky & Kahneman, 1974)
2. 손실회피 (Kahneman & Tversky, 1979)
3. 부여효과 (Thaler, 1980)
4. 상호성 (Cialdini, 2001)
5. 프레이밍 (Kahneman, 1984)
6. 사회적 증거 (Cialdini, 2001)
7. 전술적 공감 (Chris Voss, 2016)
8. BATNA (Fisher & Ury, 1981)

## 이모지/특수문자 사용 금지
- 전체 문서에서 이모지 미사용
- 대신 [핵심], [심리], [멘트], [참고] 등 태그 사용

## PDF 디자인 기준
- 본문: 13pt (Noto Sans KR)
- 테이블: 12pt
- 표지 정보: 24px 중앙정렬
- 페이지 분리: 각 h1 전에 자동
- 잘림 방지: 테이블, 인용블록 page-break-inside: avoid
