# 어반브릭스 오피스텔 데이터 수집 기준

## 1. 단지 개요
*   **단지명:** 어반브릭스 (오피스텔)
*   **세대수:** 462세대 / 총 3개 동 (STUDIO 1동, STUDIO 2동, STUDIO 3동)
*   **특징:** 유니시티 아파트와는 구분되는 오피스텔이나, 데이터베이스의 일관성과 사용자의 직관적인 검색(아파트 평형 체감)을 위해 면적 매핑 시 **전용면적 기준 아파트 평형(25평 등)**을 강제로 적용합니다.

## 2. 평형-면적 매핑 테이블 (아파트 평형 표준화)
네이버 부동산 등 외부에서는 '계약면적'을 기준으로 "43평형" 등으로 부르지만, 우리 **내부 시스템(`area_mapping.py`) 및 NotebookLM에서는 전용면적을 기준으로 아파트와 동일한 평형으로 강제 변환**합니다.

| 네이버 타입(계약/전용) | 분양(광고) 평형 | **시스템 내 강제 매핑 평형** | 세대수 | 비고 |
|:---|:---:|:---:|:---:|:---|
| **77D㎡** (77.28/22.64) | 23평(D) | **10평** | 72 | 원룸형 |
| **106C㎡** (106.58/38.56) | 32평(C) | **17평** | 72 | 투룸형 |
| **143A-1㎡** (143.34/59.50) | 43평(A) | **25평** | 106 | 주력평형 (아파트 25평형 실면적) |
| **143A-2㎡** (143.53/59.50) | 43평(A) | **25평** | 106 | 주력평형 |
| **144B㎡** (144.47/59.60) | 43평(B) | **25평** | 106 | 주력평형 |
| **총계** | | | **462** | |

> **⚠️ 주의사항**
> - 실거래가 조회, 네이버 크롤링 등에서는 `area_m2 = 143` (공급) 이나 `59.5` (전용) 값이 들어오지만, `area_mapping.py` 의 `validate_area(..., complex_name='어반브릭스')` 함수를 통과하면 모두 **"25평"**으로 반환됩니다.
> - 따라서 시스템에 기록된 모든 어반브릭스의 59㎡ 거래 건은 "43평"이 아닌 "25평"으로 남게 되며, NotebookLM에도 "25평 실거래가 3.1억"으로 학습됩니다.

## 3. 향후 수집 파이프라인 연동 계획
1. **실거래 이력 수집:** `record_trade.py` 등에서 단지명을 "어반브릭스"로 넘겨주면, 평형 검증 함수가 자동으로 오피스텔 25평/17평/10평으로 필터링 및 검증(`2.5억~4억` 정상 통과)합니다.
2. **보유비용(세금) 산정:** 오피스텔 취득세(4.6%) 적용.
3. **NotebookLM 전용 소스:** `10_어반브릭스_오피스텔_분석자료.md` 파일에 "43평"이라는 말을 감추고 25평으로 통일하여 업로드됨.
